# Oxidly Cloud Platform Roadmap (v2.0.0)

Oxidly is the enterprise SaaS frontend for the FreeRadical ecosystem.

## Core Feature Requirements
- **Multi-tenant SaaS**: Support for multiple domains and user accounts on a shared infrastructure.
- **Site Management**: Site creation, domain setup (CNAME validation), and multi-site management.
- **Theme & Plugin System**: ZIP-based uploads, preview system, and versioning.
- **Role-Based Access Control (RBAC)**: Admin, Editor, Author, Contributor, Subscriber roles.
- **AI-Powered CRM**: Automated sentiment analysis, predictive analytics, and automated messaging.

## Implementation Phases
### Phase 5: core Frontend (v2.0.0)
- Handlebars template system.
- Auth flow (Signup/Login/Email Verification).
- Dashboard shell and API client for backend.

### Phase 6: AI E-commerce UI (v2.1.0)
- AI Store Architect (Natural Language Setup).
- AI Image Studio (Background removal, scene generation).
- Conversational Shopping Assistant widget.
- Demand forecasting dashboard.

### Phase 7: Plugin Marketplace (v2.2.0)
- **Plugin Discovery UI**: Marketplace for third-party extensions.
- **Submission Workflow**: Review and approval system for developers.
- **Versioning & Updates**: automated update delivery.
- **Revenue Sharing**: Payment and commission tracking.

### Phase 8: Enterprise Features (v2.3.0)
- **Multi-tenant Architecture**: Enterprise-grade isolation.
- **Team Collaboration**: Advanced role permissions and sharing.
- **SSO Integration**: SAML/OIDC support.
- **White-labeling**: Custom branding options for agencies.
- **Audit Logs**: Comprehensive activity tracking.

### Phase 9: Infrastructure & Scale (v2.4.0)
- **Multi-region Deployment**: Gloabl availability.
- **CDN Integration**: Automated static asset delivery.
- **Redis Cluster**: Scalable caching.
- **Database Sharding**: Support for massive multi-tenancy.
